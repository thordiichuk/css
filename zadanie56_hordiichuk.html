1. składnia teoretyczna definiowania klasy

    selektor.klasa { cecha: wartość }

  2. Podaj opis dla Klasa "cecha" oraz "wartość"

    Klasa to wartość atrybutu class="..." nadanego selektorowi z poziomu języka (X)HTML.
    "cecha" oraz "wartość" określają atrybuty elementu nadane poprzez style.

  3. Jaki znacznik może być selektorem? Trzy przykłady selektorów dla klas

    Selektorem może być dowolny znacznik,
    1. p → akapit,
    2. h1 → tytuł,
    3. td → komórka tabeli i inne

  4. Podaj uwagi na temat nazwy klas:

    Nazwą może być to dowolny pojedynczy wyraz,, nie może on zawierać
    1. znaków: spacji, kropki, przecinka, dwukropka, pytajnika, nawiasów, znaku równości, plusa itp.
    Może zawierać
    1. litery (A-Z, a-z)
    2. cyfry (0-9)
    3. myślniki ("-")
    4. podkreślniki ("_")

    Nazwa nie może zaczynać się od cyfry oraz myślnika
    Gdy chcemy użyć znaków "zakazanych" znaków, należy w deklaracji klasy poprzedzić je odwróconym ukośnikiem "\", np. reguła: selektor.B\&W\? { cecha: wartość } odpowiada klasie: class="B&W?".

  5. Jakie jest zastosowanie klas?

    Deklaracja klasy jest przydatna szczególnie, jeśli na wielu stronach serwisu znajdują się elementy mające takie samo formatowanie, a dodatkowo nie można dla nich posłużyć się selektorem typu, ponieważ na stronie znajdują się inne znaczniki tego samego rodzaju co wybrany element, ale nie chcemy, aby i one otrzymały takie samo formatowanie.

  6. Kiedy ma zastosowanie lista klas?

    Jeśli chcemy do pojedynczego elementu przypisać kilka różnych klas, należy posłużyć się poleceniem:

  7. Składnia teoretyczna listy klasy oraz uwagi

    <selektor class="klasa1 klasa2 klasa3...">...</selektor>

    Uwagi:
    • przy czym klasy: klasa1, klasa2, klasa3 muszą być zdeklarowane w wewnętrznym, zewnętrznym lub importowanym arkuszu stylów.
    • Kolejność wpisywania nazw klas w wartości atrybutu class="..." nie ma znaczenia na ostateczny sposób formatowania elementu. Jeżeli kilka przypisanych klas odnosi się do tej samej własności, o wyniku decyduje zasada kaskadowości.

  8. Czym są podzbiory klas. Składnia teoretyczna listy klasy

    Istnieje możliwość określenia tzw. podzbioru klas, czyli podania, jakie klasy musi mieć jednocześnie przypisane element, aby wybrana reguła CSS miała do niego zastosowanie. Na przykład po wpisaniu:
    selektor.klasa1.klasa3 { cecha: wartość }

    zmieniony zostanie sposób formatowania elementu:
    <selektor class="klasa1 klasa2 klasa3">...</selektor>
    Kolejność podawania klas - zarówno w regule CSS, jak i w wartości atrybutu class="..." - nie ma znaczenia.

  9. Co jest sygnałem (przesłanką), że należy zastosować klasy?

    Jeśli jakiś element, o specyficznym rodzaju formatowania, wstępuje przynajmniej raz na niektórych stronach Twojego serwisu (może to być np. główny tytuł strony), jest to znak, że dobrze byłoby umieścić w zewnętrznym arkuszu stylów deklarację jego klasy.

  10. Kiedy i z jakiego powodu powinny być planowane klasy dla wykonywanej strony?

    Im więcej deklaracji klas - odnoszących się do powtarzalnych elementów - umieścisz w zewnętrznym arkuszu stylów, tym szybciej przebiegnie późniejsza ewentualna modyfikacja stron!

  11. Jaki jest cel stosowania komentarzy podczas definiowania klas?

    Po jakimś czasie w Twoim arkuszu stylów może znaleźć się tyle klas, że zapomnisz, do jakich elementów się one odnosiły.

  12. Podaj konkretny przykład definiowania klasy oraz jej użycia(wywołanie).

    definicja arkusza styli

    p.przyklad_klasa { color: red }

    wywołanie

    <p class="przyklad_klasa">To jest akapit.</p>

  13. Omów klasa donosząca się tylko do jednego selektora wraz z przykładem.

    Podobnie jak w przypadku identyfikatora, tak zdefiniowana klasa może zostać użyta tylko w tym selektorze, do którego została przypisana, np.: definicja arkusza styli

    p.moja{ color:green; font-size:14pt; }

    wywołanie

    Powinna zostać wywołana w znaczniku <p></p> w następujący sposób: <p class=”moja”>Treść akapitu</p>

  14. Jaki jest cel stosowania klasy uniwersalnej. Jakie jest znaczenie znaku „*” oraz kiedy można nie stosować tego znaku. Podaj przykład (definicja, wywołanie).

    W przypadku stosowanie klas, często podczas ich definiowania, nie wiemy jeszcze, w stosunku do jakich znaczników będziemy chcieli je później użyć. Wtedy przydatny okazuje się selektor uniwersalny, który pozwala przypisać własności klasy do wszystkich elementów, którym zostanie ona nadana, bez względu na typ znacznika czyli użycie jest możliwe w różnych znacznikach. Przykład:
    definicja arkusza styli

    *.przyklad_uniwersalna { color: red }

    Każdy element, któremu nadamy klasę przyklad_uniwersalna, będzie miał kolor czerwony. W tym przypadku gwiazdkę (*) w regule stylu można pominąć:

    .przyklad_uniwersalna { font-size:12 pt; color: red }

    Zwróć uwagę na fakt występowania kropki po selektorze lub przed nazwą klasy. Jest to element obowiązkowy i bez tego dalsza praca nie miałaby większego sensu.

    Przykład
    definicja arkusza styli

    .moja { color:red; font-size:12pt; font-style:italic; }
    wywołanie
    Wywołanie klasy tak określonej klasy również odbywa się za pomocą atrybutu class.

    <p class=”moja”>Przykład akapitu z określoną klasą</p> <ol class=”moja”> <h1 class=”moja”>Przykład nagłówka z określoną klasą</h1>
